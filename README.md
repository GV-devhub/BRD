# 🌐 Aidivehub – Learning Documentation Blog  
A centralized platform for documenting daily learning, AI tool usage, SQL/Excel practice, analytics workflows, and productivity insights.

This repository contains all project documentation, diagrams, requirements, and supporting materials used to build the Aidivehub blog as defined in the official Business Requirements Document (BRD).

🔗 **Live Blog:** https://aidivehub.blogspot.com/

---

## 📘 Project Overview

Aidivehub is a self‑managed learning and documentation blog created and maintained by **Admin**.  
The platform selected is **Blogger**, based on a structured evaluation of Wix, WordPress, and Blogger.

The blog serves as:

- A personal learning repository  
- A professional portfolio  
- A future‑ready platform for monetization (AdSense, affiliates)  
- A centralized space to organize scattered notes and insights

### **2. Platform Evaluation Notes (Wix vs WordPress vs Blogger)**  
As per BRD Appendix:

- Wix → Easy but limited  
- WordPress → Powerful but requires hosting  
- Blogger → Free SSL, zero cost, Google ecosystem  

**Final Selection:** **Blogger**

**File:** `/docs/Platform_Evaluation_Notes.md`

---

### **3. Process Flow (As‑Is & To‑Be)**  
From BRD Sections 11–14:

**As‑Is:**  
Learning scattered across notes, files, and devices.

**To‑Be:**

**High‑Level Flow:**
